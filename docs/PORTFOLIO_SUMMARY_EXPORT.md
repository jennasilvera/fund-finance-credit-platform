# Portfolio Summary Export

This document describes the portfolio monitoring summary CSV generated by the Fund Finance Credit Underwriting & Portfolio Monitoring Platform.

> This project uses simulated data only and is intended for portfolio demonstration purposes.

---

## 1. Purpose

The portfolio summary export creates a structured CSV view of facility-level portfolio monitoring information.

The goal is to provide a reviewer-friendly output that can be opened in Excel, uploaded into reporting tools, or used as a portfolio surveillance snapshot.

---

## 2. CLI Usage

Run:

```bash
fund-finance export-portfolio-summary
```

The output is written to:

```text
data/outputs/portfolio_summary.csv
```

---

## 3. Output Fields

The CSV includes:

| Field | Description |
|---|---|
| facility_id | Facility identifier |
| fund_id | Fund identifier |
| fund_name | Fund name |
| facility_type | Subscription, NAV, or hybrid |
| commitment_amount_usd | Total facility commitment |
| outstanding_amount_usd | Current drawn exposure |
| max_ltv_pct | Maximum LTV threshold |
| risk_score | Latest credit risk score |
| credit_rating | Latest credit rating |
| recommendation | Latest credit recommendation |
| open_monitoring_events | Number of unresolved monitoring events |
| escalation_events | Number of unresolved escalation events |

---

## 4. Review Use Case

A reviewer can use the export to quickly identify:

- Facility exposure
- Facility type
- Credit score
- Recommendation
- Open monitoring events
- Escalation activity

This makes the project feel more like a portfolio monitoring workflow rather than only a facility-level calculator.

---

## 5. Demo Workflow

The export is included in the main demo workflow:

```bash
make demo
```

Near the end of the demo, the workflow exports the portfolio summary CSV and lists generated credit memo PDFs.
